# コーディングルール (Coding Rules)

本プロジェクトにおける開発・実装では、以下のコーディングルールを厳守してください。

1. **コメントは日本語で記述すること**
   ソースコード内のすべてのコメントは日本語で記述し、プロジェクトメンバー全員がスムーズに理解できるようにしてください。

2. **コメントは「意味（Intent）」を説明すること**
   コードを見ればわかる「実装の詳細（How）」ではなく、なぜそのように実装したのかという「意図や意味（Why/What）」を説明した内容を記述してください。

3. **ヘッダコメントの必須化**
   クラス、関数、メソッドの定義前（ヘッダ）には、必ずその役割や引数・戻り値を説明するコメント（JSDoc等）を記載してください。

4. **1関数（メソッド）は50〜100ステップ程度に制限**
   視認性と保守性を担保するため、1つの関数やメソッドが肥大化しないよう心がけてください。長くなる場合は適切に処理を分割・別関数に切り出してください。

5. **命名規則（一般的な英語の使用と長さの制限）**
   関数名や変数名は、標準的で一般的な英語を使用してください。また、過度に長くなりすぎないよう、簡潔かつ意味が伝わる命名を心がけてください。

6. **`var` の使用禁止（`let` および `const` を使用）**
   変数の宣言にはブロックスコープを持つ `let` または `const` のみを利用し、古い `var` キーワードは例外なく使用を禁止します（基本は `const` を優先してください）。
