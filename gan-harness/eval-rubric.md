# Evaluation Rubric: ZenSync

このプロジェクトは、ブラウザの同期エンジンをプログラムから制御し、ブックマークの重複を物理的に防止することを目的としています。評価者は以下の基準に基づいて、実装の品質を判定してください。

## 1. Design Quality (weight: 0.3)
- [ ] **ビジュアル状態表示**: ダッシュボード上で、各ブラウザの同期状態（ON/OFF/移行中）が明確に区別できるか？
- [ ] **UXの滑らかさ**: ブラウザが終了・再起動される際、ユーザーが「何が起きているか」を把握できる適切なトースト通知やインジケーターがあるか？
- [ ] **UIのプロフェッショナル性**: AI生成特有の過度な装飾が避けられ、機能的かつモダンなデザインになっているか。

## 2. Originality & Innovation (weight: 0.2)
- [ ] **設定ファイルハックの精度**: 単なるテキスト置換ではなく、JSONとしてPreferencesを解析・更新しているか？
- [ ] **対象キーの妥当性**: `sync.bookmarks` だけでなく、`sync.keep_everything_synced` などの依存キーを適切に処理しているか。

## 3. Craft & Robustness (weight: 0.3)
- [ ] **同期の確実な無効化**: ブラウザ起動直後の設定画面（`chrome://settings/syncSetup/advanced`）で、実際にブックマーク同期がOFFになっているか？
- [ ] **アトミック性**: `Bookmarks` の書き込みと `Preferences` の書き込みがセットで行われ、途中で失敗した場合に両方がロールバックされるか？
- [ ] **クリーンアップ**: クラッシュフラグの除去が確実に行われ、再起動時に「異常終了しました」というダイアログが出ないか。

## 4. Functionality & User Flow (weight: 0.2)
- [ ] **マルチブラウザ対応**: Chrome, Edge, Brave のすべてで同期のON/OFF制御が機能するか。
- [ ] **自動復帰ロジック**: 保存完了後、または次回のアプリ起動時に、同期が自動的に ON に戻る「ブーメラン」機能が動作するか。
- [ ] **エラーハンドリング**: Preferencesファイルへのアクセス権限エラーや、ブラウザプロセスの終了失敗を適切に検知し、ユーザーに通知できるか。

---

## 合格基準 (Passing Criteria)
- スコア 70/100 以上。
- 致命的なバグ（Preferencesが壊れてブラウザが起動しなくなる等）がないこと。
- Chrome/Edge/Brave のうち少なくとも2つ以上で完全に動作すること。
